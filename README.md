# modeling-lab
A hands-on lab for learning how to model real problems with math, data, and small machine learning/deep learning models.

## Structure

- `ml/traditional-ml/`: the main track for classical machine learning, from interpretable models to statistical learning ideas.
- `ml/deep-learning/`: neural-network and representation-learning labs.
- `math/`: applied mathematics labs, organized by topic and lab number.

Deep learning labs:

- `ml/deep-learning/parameter-efficient-finetuning/lab1`: LoRA fine-tuning with Qwen3.5-2B-Base.
- `ml/deep-learning/algorithmic-grokking/lab1`: small Transformer grokking modular addition.

The intended ML learning track should start with traditional, interpretable methods, then move toward neural networks:

1. decision trees and impurity criteria
2. linear/logistic models and regularization
3. bias-variance, validation, and model selection
4. ensembles such as random forests and gradient boosting
5. simple interpretable neural networks
6. deeper representation-learning cases
